1 @section intro Introduction
2 
3 Squash is an abstraction layer which provides a single API to access
4 many compression libraries.
5 
6 For a high-level overview of how to use Squash, please see the
7 [Squash User Guide](@ref md_user-guide).
8 
9 @subsection bindings Bindings
10 
11 Squash is written in C and has minimal hard dependencies, meaning it
12 should be straightforward to create bindings for many languages. In
13 order help with testing, bindings are generally distributed with
14 Squash.
15 
16 Squash currently supports the following languages:
17 
18 - C
19 - Vala
20 
21 We hope to add more soon. If you're interested in helping, please
22 contact us!
23 
24 @subsection plugins Plugins
25 
26 The actual integration with individual compression libraries is done
27 through plugins which can be installed separately from Squash itself
28 and are not loaded until they are required. This allows Squash
29 consumers to utilize a great many compression algorithms without
30 rewriting code or unnecessary bloat. Plugins need not be distributed
31 with Squash, though it is preferred.
32 
33 Squash currently contains plugins for the following libraries:
34 
35 - [bzip2](@ref md_bzip2)
36 - [lzma](@ref md_lzma)
37 - [lzo](@ref md_lzo)
38 - [snappy](@ref md_snappy)
39 - [zlib](@ref md_zlib)
40 
41 We hope to add more soon. If you're interested in helping, please
42 contact us!
