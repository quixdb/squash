1 # Building Squash
2 
3 Note: [Squash currently does not support
4 Windows](https://github.com/quixdb/squash/issues/86).
5 
6 Squash currently requires a system copy of liblzma (from xz utils).
7 We're working on removing this limitation, but in the meantime there
8 is probably a package available for your system… xz-devel on Fedora,
9 liblzma-dev on Debian/Ubuntu, liblzma on FreeBSD, etc.

11 ## Building from git
12 
13 If you are building from a release, skip this section.
14 
15 Squash makes very heavy use of git submodules to pull in code from
16 other projects, mostly for the libraries used by each plugin. When
17 possible Squash will use system libraries instead of its own copy, but
18 only a relatively small number of plugins use libraries which are
19 typically packaged by distributions.
20 
21 If you are on a UNIX-like system with bash, you can simply call the
22 `autogen.sh` script (located in the top level of the Squash sources)
23 just as you would call the `configure` script. If not you can
24 update the submodules manually—from the top level of the Squash
25 sources:
26 
27 ~~~{.c}
28 git submodule update --init --recursive
29 ~~~
30 
31 `autogen.sh` simply invokes this command then calls `configure` with
32 the arguments you passed to it.
33 
34 Once you have all the submodules your source tree should be in
35 basically the same state as a release (except you still have all the
36 git data), so proceed to the next section.
37 
38 ## Building from a release
39 
40 If you are on a UNIX-like system (basically non-Windows) with bash
41 (basically non-BSD), simply run the `configure` script with the same
42 arguments you would pass to the `configure` script of any
43 autotools-based project:
44 
45 ~~~{.c}
46 ./configure
47 ~~~
48 
49 This will translate the arugments you pass to the CMake versions and
50 invoke CMake.
51 
52 For a list of supported arguments, pass `--help`. If you don't have
53 bash, you'll have to call CMake manually:
54 
55 ~~~{.c}
56 cmake .
57 ~~~
58 
59 At this point, all you need to is call `make` likely `make install`,
60 just as you would for any other project.
